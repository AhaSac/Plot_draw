
- **总行数**：29（包含表头）
- **有效数据行**：28 行

## 2. 关键常量定义

在 `cylinder_plot_common.py` 中：
```python
INITIAL_LHS_LAST_CASE = 27  # 第27个案例是初始 LHS 的最后一个
```

这意味着：
- `case_index ≤ 27` → 属于 "LHS screening" 阶段
- `case_index > 27` → 属于 "GPR-guided verification" 阶段

## 3. 数据加载流程（load_data() 函数）

```python
def load_data():
    df = pd.read_csv(CSV_PATH)
    
    # 提取基准设计（第2行 CSV 数据）
    baseline = df.iloc[0].copy()        # case="0", case_index=0
    
    # 提取所有优化设计（第3行 CSV 数据及之后）
    design = df.iloc[1:].copy()         # G01-G27 及之后的所有案例
    
    # 为 design 添加派生字段
    design["feasible"] = (length_ok) & (pressure_ok) & (eigen_ok)
    design["mass_reduction"] = 1.0 - design["total_mass"] / baseline["total_mass"]
    design["stage"] = np.where(
        design["case_index"] <= 27, 
        "LHS screening",
        "GPR-guided verification"
    )
    
    # 从可行设计中选出最优解
    best = design[design["feasible"]].sort_values("total_mass").iloc[0]
    
    return df, baseline, design, best, pressure_limit
```

## 5. ⚠️ 删除前28行会导致的错误

### 当前情况（假设你选了第3到29行，共27行）：
选中的行范围包括：
- CSV 第3行：G01（case_index=1）
- ...
- CSV 第29行：G27（case_index=27）

### 删除后的后果

#### 1️⃣ **数据丢失 - 初始 LHS 阶段的 27 个样本全部消失**
```
删除前：baseline(1) + G01-G27(27) = 28 行有效数据
删除后：baseline(1) + 其他(0) = 严重数据不足
```

#### 2️⃣ **stage 分类失效**
- 删除后没有任何设计满足 `case_index <= 27`
- 所有剩余设计都被标记为 "GPR-guided verification" 阶段
- `plot_01_optimization_summary.py` 等依赖阶段分类的图会显示不正常

#### 3️⃣ **GPR 模型拟合失败**
```python
# 交叉验证需要样本数 >= n_splits（默认5）
def cross_validated_gpr_predictions(design, RESPONSE_COLS):
    # 如果 design 只有 1-2 行，交叉验证会直接报错或产生垃圾结果
    kfold = KFold(n_splits=5, shuffle=True, random_state=42)
    for train_idx, test_idx in kfold.split(design):
        # 训练集几乎为空 → 模型无法训练
```

#### 4️⃣ **参数响应关系图无法绘制** 
- `plot_04_parameter_response_grid.py` 会产生稀疏的散点图
- 无法显示设计空间的趋势关系

#### 5️⃣ **平行坐标图失效**
- `plot_05_parallel_coordinates.py` 需要显示多条平行线
- 如果只有 1-2 个设计，图形无法表现参数的变化规律

#### 6️⃣ **参数演化热图无法绘制**
- `plot_06_parameter_evolution_heatmap.py` 需要按 case_index 的时间序列
- 数据太少无法形成有意义的热图

## 6. 🔴 具体的运行时错误示例
